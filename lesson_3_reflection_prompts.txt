Cuando querrias usar un repositorio remoto en vez de trabajar localmente?

Cuando quisiera colaborar con otras personas en un proyecto o si quisiera respaldar mi proyecto en la nube.

Por que quisieras hacer los pull de los cambios manualmente en vez de que Git los tenga actualizados automaticamente con el repositorio remoto?

Porque me gusta decidir cuando subir los cambios o descargar los que ya esten en la nube para evitar conflictos.

Describe la diferencia entre forks, clones y branches. Cuando deberias usar uno u otro?

- Clone se utiliza para copiar un repositorio a la maquina local. Esto es indiferente si es un repositorio publico del cual no eres dueño, es el tuyo propio o es un fork de otro proyecto.
- Forks se utiliza para copiar un repositorio del cual no tienes permisos de escritura sobre él. Se utiliza para clonar un repositorio y poder colaborar directamente en Github, permitiendo poder copiarlo de una forma mas sencilla.
- Branches se utiliza cuando se desea trabajar en un feature aparte sin tocar el codigo principal, independientemente si es un fork o el propio proyecto. 

Cual es el beneficio de tener el ultimo estado conocido del repositorio en forma local?

Que se tiene un respaldo mas actual de lo que se estuvo desarrollando y poder trabajar sobre ello y con esto evitar la menor cantidad de conflictos posibles al reintegrar los cambios nuevamente.

Como colaborarias sin usar git o Github? Que seria mas facil y mas dificil?

Sin Git ni Github seria mucho mas complicado integrar cambios entre diferentes personas. Usando estas dos herramientas todo es mas rapido y mas facil que haciendolo de forma manual.

Cuando quisieras hacer cambios en una rama separada en vez de hacerlo directamente sobre master? Que beneficios trae esta manera de trabajo?

Utilizaria una rama aparte siempre que quisiera hacer algun cambio en el trabajo actual, sobre todo si lo que quiero es evitar dañar el trabajo que ya ha sido probado y aprobado por varias personas. De esta manera se mantiene una forma de trabajo mas limpia y segura que mezclar todo de una sola vez. Tambien me permitiria utilizar las ramas para compartir cambios a otras personas sin que la rama principal, master, se vea perjudicada.
